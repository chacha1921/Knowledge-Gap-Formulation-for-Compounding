Slide 2 of 7 — Mechanism 1: How the VLM Sees an Image Source: Dosovitskiy et al., An Image is Worth 16×16 Words — ICLR 2021

Vision Transformer (ViT) — The Foundation

1
Patch Division

Image split into non-overlapping square patches (e.g. 16×16 px each)

→
2
Flatten + Project

Each patch flattened and projected into the model's embedding dimension

→
3
Positional Encoding

Learnable positional embeddings added to preserve spatial order within the patch grid

→
4
Unified Sequence

Patch embeddings + text token embeddings → one sequence, one attention mechanism

Key insight: Image patches and text words are the same type of object — vectors in the same embedding space. When the prompt says "extract the revenue column," the word tokens for "revenue" attend directly to the patch embeddings covering that region. There is no separate vision pathway.
Within a tile — spatial layout preserved

Positional encodings are learned across the patch grid. The model knows patch (3,2) is to the right of patch (3,1).

Across tiles — NO shared encoding

Tiles are independent crops. A patch at position (0,0) of tile 2 has no positional relationship to any patch in tile 1.

Slide 3 of 7 — Mechanism 2: What detail: high Actually Does Source: OpenAI Vision Documentation, 2024

The 4-Step Tiling Algorithm

StepActionResult
1Resize to fit within 2048 × 2048 pxPreserves aspect ratio
2Scale so shortest side = 768 pxStandardises resolution
3Tile image in 512 × 512 windowsEach tile is independent
4Token cost: 85 + (170 × tiles)Base overview + per-tile cost

Concrete Calculation — Scanned A4 at 150 DPI

import math

def vision_tokens_high(w, h):
    if max(w,h) > 2048: s = 2048/max(w,h); w,h = int(w*s), int(h*s)
    if min(w,h) > 768:  s = 768/min(w,h);  w,h = int(w*s), int(h*s)
    tiles = math.ceil(w/512) * math.ceil(h/512)
    return {"dims":(w,h), "tiles":tiles, "tokens":85+170*tiles}

print(vision_tokens_high(1240, 1754))  # A4 @ 150 DPI

Slide 4 of 7 — Why Wide Tables Fail The failure mode I could not predict — now I can

Tile Boundary = Row Continuity Break

Each 512 × 512 tile at 150 DPI covers ~3.4 × 3.4 inches of the original page.
An A4-wide table (8.27 inches) spans 2–3 tiles.
Tiles are independent crops — no shared positional encoding across tile boundaries.
A table row that straddles a boundary appears as two disconnected groups of patches.
The model cannot reconstruct row continuity without a structural signal that does not exist.

Tables that succeed

PropertyWhy it works
Width < 3.4 inchesFits within one tile — full spatial layout preserved
Dense, single-columnCells stay within tile boundaries
Center of pageNot split by edge tiling

Tables that fail

PropertyWhy it breaks
Width > 3.4 inchesRow straddles tile boundary — continuity lost
Near page edgeContent cut by margin tile — partial rows
Multi-column wideColumns in different tiles with no linking signal
Now I can predict failure: if a table's width exceeds one tile width (~3.4 inches at 150 DPI), Strategy C will likely hallucinate or miss rows. This is checkable before the API call.
Slide 5 of 7 — The Fix: SmartBBoxCropper 15-line OpenCV preprocessing — committed to Document Intelligence Refinery

The Insight from the Explainer

If I crop the page to its content bounding box before sending to the VLM, the whitespace margins are removed, the effective image is smaller (fewer tiles), and tables are more likely to fall within a single tile instead of straddling a boundary.

import cv2, numpy as np

def smart_bbox_crop(img_bytes: bytes, padding: int = 20) -> bytes:
    nparr  = np.frombuffer(img_bytes, np.uint8)
    img    = cv2.imdecode(nparr, cv2.IMREAD_COLOR)
    gray   = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)
    _, thr = cv2.threshold(gray, 250, 255, cv2.THRESH_BINARY_INV)
    coords = cv2.findNonZero(thr)
    if coords is None: return img_bytes           # blank page
    x, y, w, h = cv2.boundingRect(coords)
    ih, iw = img.shape[:2]
    x, y = max(0, x-padding), max(0, y-padding)
    w, h = min(iw-x, w+2*padding), min(ih-y, h+2*padding)
    _, buf = cv2.imencode('.png', img[y:y+h, x:x+w])
    return buf.tobytes()

# One line added in _extract_gemini and _extract_ollama:
img_bytes = smart_bbox_crop(img_bytes)   # crop before VLM call
